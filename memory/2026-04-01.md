# 2026-04-01 - Bugfix: Übungen/Workouts nicht sichtbar nach Google Sync

## Problem
Nach Google-Login und Daten-Restore wurden keine Übungen und Workouts angezeigt, obwohl die Daten synchronisiert wurden.

## Ursache
In `app.js` wurde nach erfolgreichem Restore `window.location.reload()` aufgerufen, aber `sessionStorage.setItem('restoreAttempted', 'true')` blieb im sessionStorage gespeichert. Nach dem Reload war das Flag immer noch auf `'true'`, wodurch der Code-Block zum Laden der Übungen und Workouts übersprungen wurde.

## Fix Teil 1: Restore-Flag
In `webapp/public/app.js` wurden zwei Stellen geändert:

1. Automatisches Restore - vor `window.location.reload()`:
```javascript
sessionStorage.removeItem('restoreAttempted');
window.location.reload();
```

2. Manueller Restore mit Token - gleiche Änderung

### Fix Teil 7: Backup Route `/api/backup/drive`

### Fix Teil 8: Google Token aus JWT speichern
**Problem:** Backup gibt 400 "Nicht mit Google verbunden"

**Ursache:** Die User-Tabelle hat `google_access_token = null` für bestehende Accounts. Der Server holt das Token aus der DB, nicht aus dem JWT.

**Lösung:** Der `authenticateJWT` Middleware speichert jetzt automatisch das Token aus dem JWT in die DB:
```javascript
const jwtGoogleToken = decoded.googleAccessToken;

// Versuche Google Token zu laden
if (!err && row) {
  // Wenn JWT Token vorhanden aber DB null → speichern
  if (jwtGoogleToken && !row.google_access_token) {
    db.run('UPDATE users SET google_access_token = ? WHERE id = ?', 
      [jwtGoogleToken, decoded.userId]);
    req.user.googleToken = jwtGoogleToken;
  } else {
    req.user.googleToken = jwtGoogleToken || row.google_access_token;
  }
}
```

### Fix Teil 9: Wiederholtes Restore verhindern
**Problem:** Nach erfolgreichem Restore wurde immer und immer wieder versucht zu restoren.

**Ursache:** Das Flag `restoreAttempted` wurde nach erfolgreichem Restore gelöscht, damit `loadExercises()` aufgerufen werden kann. Aber dann wurde beim nächsten Seitenaufruf wieder versucht zu restoren, weil das Flag fehlte.

**Lösung:** Zusätzliches Flag `alreadyRestored` eingeführt:

```javascript
const restoreAttempted = sessionStorage.getItem('restoreAttempted');
const alreadyRestored = sessionStorage.getItem('alreadyRestored');

// Wenn schon restored in dieser Session, überspringe
if (alreadyRestored === 'true') {
    console.log('✅ Bereits restauriert, überspringe Restore');
} else if (!restoreAttempted) {
    // Versuche Restore...
    if (restoreResult) {
        sessionStorage.setItem('alreadyRestored', 'true');
        window.location.reload();
    }
}
```

Beim Logout werden beide Flags gelöscht:
```javascript
sessionStorage.removeItem('restoreAttempted');
sessionStorage.removeItem('alreadyRestored');
```
